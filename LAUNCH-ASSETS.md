## 4. Reddit Post

**Title:** Built a UBI token on Base — no ICO, no staking, no team allocation. Just a daily free claim. Looking for feedback.

**Body:**

Hey all — I deployed an ERC-20 token on Base that does one thing: lets anyone claim 100 tokens per day for free. That's the whole project. No monetization layer built on top.

**Tokenomics:**
- Total supply: 1,000,000,000 KORPO
- Daily claim: 100 KORPO per wallet per 24h
- Burn: 0.5% on any transfer exceeding 100 KORPO
- Team allocation: 0
- Admin timelock: 24 hours on any contract changes

**Why build this?**
Most "UBI" tokens are wrappers for staking schemes or have team pools that dwarf user claims. I wanted to see if the simplest possible model works — equal access, zero cost, transparent contract. The burn mechanic means tokens slowly become more scarce as the ecosystem gets used.

**What are the risks?**
- The token may have zero monetary value forever. That's fine — this is an experiment, not an investment.
- Contract risk — it's been tested (55 unit tests) but audited would be better. Review the code yourself.
- Sybil resistance is limited — one wallet per claim, but nothing stops someone from spinning up Many wallets.
- Admin functions exist but are behind a 24h timelock.

**Contract:** `0xF970c93D00De94786F6fdABBc63180da1D981bc7`
**Network:** Base mainnet
**Site:** https://korpo.pro

Not asking anyone to buy anything — there's nothing to buy. Would appreciate technical feedback on the contract design and tokenomics.
